### 2. 主题配置 API

**文件位置**: `src/app/[locale]/(admin)/admin/api/theme-config/route.ts`

**API 接口**:

#### GET /admin/api/theme-config
读取主题配置
- 参数：`?theme=default`
- 返回：主题配置对象

#### POST /admin/api/theme-config
保存主题配置
- 请求体：
```json
{
  "themeSlug": "default",
  "name": "默认主题",
  "config": { /* 配置对象 */ }
}
```
- 返回：`{ success: true, message: "主题配置已保存" }`

**功能特性**:
- 从数据库读取/保存配置
- 支持配置不存在时返回默认配置
- 异常处理和错误提示

## 使用流程

### 访问主题设置页面

1. 登录后台管理系统
2. 导航到"主题设置"页面
3. 页面路径：`/admin/theme-settings`

### 配置主题

1. **全局设置**
   - 启用主题
   - 选择配色方案
   - 设置字体样式

2. **侧边组件**
   - 勾选需要显示的组件
   - 组件会自动按排序显示

3. **首页视图**
   - 选择默认视图模式
   - 配置各视图的显示选项

4. **广告管理**
   - 展开广告位面板
   - 启用广告位
   - 选择广告类型
   - 粘贴广告代码或填写广告信息

5. **SEO 设置**
   - 填写搜索引擎验证代码
   - 启用 Sitemap 和 Robots

6. **动画效果**
   - 根据需要启用/禁用动画

### 保存配置

点击右上角"一键保存"按钮，配置会：
- 立即保存到数据库
- 前端实时生效（无需刷新）
- 显示成功提示消息

### 重置配置

点击"重置配置"按钮，会恢复所有配置为默认值。

## 技术实现

### 状态管理

使用 React useState 管理配置状态：
```typescript
const [config, setConfig] = useState(DEFAULT_THEME_CONFIG);
```

### 数据持久化

通过 API 保存到数据库：
```typescript
await fetch("/admin/api/theme-config", {
  method: "POST",
  body: JSON.stringify({ config }),
});
```

### 实时更新

配置保存在数据库后，前端通过以下方式实时生效：
1. 使用 React Context 或全局状态（可选）
